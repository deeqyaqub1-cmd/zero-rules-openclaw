üÉè What are Cards?

Cards are small chunks of knowledge your agent stores and retrieves. Think of them as sticky notes for your AI ‚Äî project details, people info, decisions, preferences. Each card has a slug (unique ID), title, body (the content), keywords (for search), and a stack (category).

üìÅ What are Stacks?

Stacks are categories that organize your cards. They help your agent find the right memory fast. Use any name you want, but here are the recommended ones:

üì¶ projects

Tech stacks, repos, architecture

üë§ people

Teammates, contacts, roles

‚öñÔ∏è decisions

Why you chose X over Y

‚öôÔ∏è preferences

Editor, tools, coding style

üîÑ workflows

Deploy steps, review processes

üìÑ general

Everything else

‚ö° API Reference

POST /api/cards?workspace=default

Create or update a card. Send JSON body with: slug, title, body, stack, keywords (array)

GET /api/cards?workspace=default

Get all cards in a workspace

GET /api/search?workspace=default&q=search term

Search cards by keywords, title, or content

DEL /api/cards?workspace=default&slug=card-slug

Delete a card by slug

All endpoints require header: X-API-Key: your_key

üêæ Using with OpenClaw

HyperStack works as an OpenClaw skill. Your agent reads the SKILL.md file and automatically knows how to create, search, and update cards.

Setup:
1. Download SKILL.md from your dashboard (Get Started tab)
2. Place in your project: skills/hyperstack/SKILL.md
3. Set env: HYPERSTACK_API_KEY=your_key
4. Your agent now has memory ‚Äî it reads the skill and handles everything

‚ùì FAQ & Troubleshooting

How does my agent use HyperStack?

Your agent calls the REST API to store and search memories. Before responding, it searches for relevant cards. When it learns something new, it creates or updates a card. This means it never forgets.

What counts as a "card"?

Any piece of knowledge ‚Äî a person's role, a project's tech stack, a decision rationale, your coding preferences. Cards are small (a few lines) and focused on one topic.

I get "Unauthorized" when calling the API

Make sure you're sending the header X-API-Key: your_key (not Authorization). Your key is in the API Key tab of your dashboard. It should start with hs_.

What fields does a card need?

Required: slug (unique ID, like "my-project"), title, body. Optional but recommended: stack (category), keywords (array of tags for search). If you POST the same slug again, it updates the existing card.

How do I delete a card?

Send a DELETE request with the slug as the id parameter: DELETE /api/cards?workspace=default&id=my-card-slug

How do I save tokens?

Without memory, agents stuff everything into the prompt (6,000+ tokens). With HyperStack, the agent searches for only the relevant card (~350 tokens). That's 94% less.

What's a workspace?

A workspace is a separate bucket of cards. Use "default" for everything, or create separate workspaces per project (Pro feature) so cards don't mix.

Can I use any AI agent?

Yes. HyperStack is a REST API. If your agent can make HTTP requests, it works ‚Äî OpenClaw, Claude Code, LangChain, CrewAI, AutoGPT, custom agents, or even a simple Python script.

Is my data safe?

Your cards are stored in a secure PostgreSQL database. Only your API key can access your data. We don't read, sell, or share your cards.

What happens if I hit 50 cards on Free?

You can still read and search cards, but you can't create new ones until you delete some or upgrade to Pro (unlimited cards).

PowerShell users: curl doesn't work!

PowerShell aliases curl to Invoke-WebRequest. Use Invoke-RestMethod instead, or run curl commands in Git Bash / WSL / CMD.
