CORE CONCEPT

🃏 What are Cards?

Cards are small, structured chunks of knowledge your agent stores and retrieves. Think of them as index cards for an AI brain. Each card has:

slug

Unique ID — same slug = update, not duplicate

title

Human-readable name for the card

keywords

Tags for exact-match search

stack

Category folder (projects, people, etc)

cardType

person, project, decision, preference, workflow, event, account

links

Connect cards by slug — build a knowledge graph

📁 Stacks Categories for your cards
📦 projects

Tech stacks, repos, architecture

👤 people

Teammates, contacts, roles

⚖️ decisions

Why you chose X over Y

⚙️ preferences

Editor, tools, coding style

🔄 workflows

Deploy steps, review processes

📄 general

Everything else

🔗 Context Graph NEW

Cards aren’t flat. They connect. Type your cards, link them together, and traverse the graph.

Card Types

person, project, decision, preference, workflow, event, account. Filter cards by type.

Card Links

Connect cards by slug. “decision-auth” links to “team-alice” and “project-webapp”.

Link Resolution

GET a card → linked cards resolved automatically with metadata.

Graph Traversal

GET /api/graph?from=slug&depth=2 — follow connections up to 3 hops.

Create or update a card. Body: {slug, title, body, stack, keywords[]}

GET /api/cards?workspace=default

List all cards in a workspace

GET /api/search?workspace=default&q=term

Hybrid search: keywords + semantic matching (pgvector)

Delete a card by slug

All endpoints require header: X-API-Key: your_key

❓ FAQ
How does my agent use HyperStack?▾
Your agent calls the REST API to store and search memories. Cards can be typed (person, decision, project) and linked to other cards by slug. Before responding, the agent searches for relevant cards. When it learns something new, it creates or updates a card. It can also traverse linked cards via the graph API to follow connections between knowledge.
What counts as a “card”?▾
Any piece of knowledge — a person’s role, a project’s tech stack, a decision rationale, your coding preferences. Cards are small (a few lines) and focused on one topic. Each card can have a type (person, decision, project, etc) and link to other cards, so your agent builds a connected knowledge graph over time.
How is this different from other memory tools?▾
Mem0 and Zep now have knowledge graphs, but they auto-extract entities using LLM calls (~$0.002/op, risk of hallucinated relationships). HyperStack lets you explicitly define cards and links — precise, zero LLM cost, instant. Plus time-travel debugging: reconstruct the entire graph at any timestamp to see what your agent knew when it made a bad decision.
I get “Unauthorized” when calling the API▾
Make sure you’re sending the header X-API-Key: your_key (not Authorization). Your key starts with hs_ and is in the API Key tab of your dashboard.
What fields does a card need?▾
Required: slug, title, body. Optional: stack (category), keywords (tags), cardType (person, project, decision, etc), links (array of card slugs to connect). If you POST the same slug again, it upserts. Use the graph API to traverse linked cards.
How does search work?▾
Hybrid search: keyword matching (instant, $0) + semantic search via pgvector (understands meaning). Ask “what auth do we use?” and it finds a card about Clerk even if “auth” isn’t a keyword — because it understands the meaning.
Can I use any AI agent?▾
Yes. Use the MCP server for Cursor/Claude Desktop/VS Code (npm install -g hyperstack-mcp), the LangGraph integration (pip install hyperstack-langgraph) for Python agents, or the raw REST API from any language. Works with Claude Code, LangChain, CrewAI, custom agents, or a simple script.
Is my data safe?▾
Your cards are stored in a secure PostgreSQL database with pgvector. Only your API key can access your data. We don’t read, sell, or train on your cards. HyperStack is self-funded — no VC pressure to monetize your data.
What happens at 10 cards on Free?▾
You can still read and search all your cards, but you can’t create new ones until you delete some or upgrade to Pro ($29/mo for 100 cards + graph traversal + visual explorer). The graph API and visual explorer are only available on paid plans.
PowerShell: curl doesn’t work!▾
PowerShell aliases curl to Invoke-WebRequest. Use Invoke-RestMethod instead, or run curl commands in Git Bash, WSL, or CMD.
